# إنشاء حساب السوبر أدمن في الإنتاج

في بيئة الإنتاج تحتاج إلى إنشاء أول حساب سوبر أدمن (أو أكثر) للدخول إلى `/super-admin`. لديك خياران:

---

## الطريقة 1: من صفحة النظام (مُفضّلة)

1. **عيّن مفتاح إعداد النظام** على السيرفر في ملف البيئة (مثلاً `.env`):
   ```env
   SYSTEM_SETUP_SECRET=مفتاح_سري_طويل_وعشوائي_مثل_أكثر_من_8_أحرف
   ```
   استخدم قيمة سرية قوية (يُفضّل مولد كلمات مرور عشوائية).

2. **شغّل التطبيق** ثم افتح في المتصفح:
   ```
   https://عنوان-موقعك.com/system
   ```

3. **أدخل المفتاح** في الحقل الظاهر (نفس قيمة `SYSTEM_SETUP_SECRET`).

4. **أنشئ الحساب**: أدخل البريد الإلكتروني، كلمة المرور (8 أحرف على الأقل)، والاسم إن رغبت، ثم اضغط إنشاء.

5. **سجّل الدخول** من الصفحة الرئيسية لتسجيل الدخول باستخدام البريد وكلمة المرور، ثم ستُوجّه تلقائياً إلى `/super-admin`.

- يمكنك العودة إلى `/system` في أي وقت (بنفس المفتاح) لإنشاء سوبر أدمن إضافيين أو لعرض القائمة.
- **مهم:** لا تشارك `SYSTEM_SETUP_SECRET` مع أحد واحتفظ به في ملف البيئة فقط على السيرفر.

---

## الطريقة 2: سكربت من متغيرات البيئة (للأتمتة)

مناسب إذا أردت إنشاء أول سوبر أدمن تلقائياً عند النشر (مثلاً في Docker أو CI).

1. **أضف المتغيرات** في بيئة السيرفر أو في `.env`:
   ```env
   FIRST_SUPER_ADMIN_EMAIL=admin@example.com
   FIRST_SUPER_ADMIN_PASSWORD=كلمة_مرور_قوية_8_أحرف_على_الأقل
   FIRST_SUPER_ADMIN_NAME=مدير النظام
   ```

2. **شغّل السكربت مرة واحدة** بعد تشغيل التطبيق وربط قاعدة البيانات:
   ```bash
   npm run create-super-admin
   ```
   أو مع تحديد المتغيرات مباشرة:
   ```bash
   FIRST_SUPER_ADMIN_EMAIL=admin@example.com FIRST_SUPER_ADMIN_PASSWORD=YourSecurePass123 npm run create-super-admin
   ```

3. السكربت **لا ينشئ** حساباً إذا وُجد بالفعل مستخدم بدور سوبر أدمن؛ يمكنك تشغيله بأمان بعد كل نشر إذا أردت.

4. بعد ذلك سجّل الدخول من واجهة تسجيل الدخول باستخدام نفس البريد وكلمة المرور للوصول إلى `/super-admin`.

---

## ملخص

| البيئة   | إنشاء السوبر أدمن |
|----------|---------------------|
| تطوير    | من `/system` بمفتاح من `.env` أو استخدام `npm run db:seed` للحساب الافتراضي |
| إنتاج    | الطريقة 1: `/system` + `SYSTEM_SETUP_SECRET` أو الطريقة 2: `npm run create-super-admin` + متغيرات البيئة |

صفحة السوبر أدمن بعد تسجيل الدخول: **`/super-admin`**.

# إذا لم يفتح السوبر أدمن بعد إنشاء الحساب

اتبع الخطوات بالترتيب:

## 1) التأكد من وجود المتغيرات وتشغيل السكربت

- في `.env` أو `.env.local` تأكد من وجود (مع قيمك الحقيقية):
  ```env
  FIRST_SUPER_ADMIN_EMAIL=البريد_الفعلي@example.com
  FIRST_SUPER_ADMIN_PASSWORD=كلمة_مرور_8_أحرف_على_الأقل
  ```
- ثبّت الحزمة ثم شغّل السكربت:
  ```bash
  npm install
  npm run create-super-admin
  ```
- يجب أن ترى في الطرفية: **"تم إنشاء أول سوبر أدمن: البريد..."**
  - إذا ظهر **"يوجد بالفعل سوبر أدمن"** فالحساب موجود؛ استخدم بريد وكلمة المرور الصحيحين لتسجيل الدخول.
  - إذا ظهر خطأ **"أضف FIRST_SUPER_ADMIN_EMAIL"** فالمتغيرات غير محمّلة؛ تأكد أن القيم في `.env` أو `.env.local` في جذر المشروع.

## 2) وجود NEXTAUTH_SECRET

- في نفس الملف (`.env` أو `.env.local`) يجب أن يكون موجوداً:
  ```env
  NEXTAUTH_SECRET=أي_نص_طويل_وعشوائي
  ```
- بدونه الجلسة قد لا تُحفظ وتُعاد إلى صفحة تسجيل الدخول.

## 3) تسجيل الدخول من صفحة الدخول

- افتح **صفحة تسجيل الدخول** (مثلاً `/login` أو الصفحة الرئيسية ثم الدخول).
- استخدم **نفس البريد** و**نفس كلمة المرور** اللذين وضعتهما في المتغيرات عند تشغيل `create-super-admin`.
- بعد النجاح يجب أن تُوجّه تلقائياً إلى **`/super-admin`**.

## 4) إذا ما زال لا يفتح

- **امسح كوكيز الموقع** (أو جرّب نافذة خاصة / متصفح آخر) ثم سجّل الدخول مرة أخرى.
- تأكد أنك لا تدخل على **`/super-admin`** مباشرة قبل تسجيل الدخول؛ ادخل من **صفحة الدخول** ثم انتقل للوحة السوبر أدمن بعد نجاح تسجيل الدخول.

## 5) إنشاء الحساب من صفحة النظام بدلاً من السكربت

- في `.env` أو `.env.local`:
  ```env
  SYSTEM_SETUP_SECRET=مفتاح_سري_طويل_8_أحرف_على_الأقل
  ```
- افتح في المتصفح: **`/system`**
- أدخل المفتاح ثم أنشئ حساب سوبر أدمن (بريد + كلمة مرور).
- سجّل الدخول من صفحة الدخول بنفس البريد وكلمة المرور.

إذا نفذت الخطوات وما زال لا يفتح، أرسل: ماذا يظهر عند تشغيل `npm run create-super-admin`؟ وهل بعد تسجيل الدخول تُوجّه إلى صفحة أخرى (مثل `/forbidden` أو تبقى في `/login`)؟
